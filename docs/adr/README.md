# Architecture Decision Records

This directory contains ADRs (Architecture Decision Records) for KRM — Kinetic Requirements Model.

ADRs document significant decisions about the framework's design, data model, and behavior. They capture context and consequences, and remain useful even when the decision is "we considered this and chose not to act."

## Format

Each ADR follows this structure:

```
# ADR-XXXX: Title

**Status:** Proposed | Accepted | Rejected | Superseded by ADR-XXXX
**Date:** YYYY-MM-DD
**Issue:** #N

## Context
What situation makes this decision necessary.

## Decision
What we are choosing to do, or not do.

## Consequences
What becomes easier, harder, or different as a result.
```
